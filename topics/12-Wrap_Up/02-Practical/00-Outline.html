Options for encoding columns

Feature Engineering plays an important role in developing machine learning models.

You have already met several aspects in Data Mining 1, and it is a major topic in Data Mining 2, where you will be introduced to more advanced techniques.

Scaling applies to numrical features, and ensures that certain features are not given to much prominence relative to their peers.

Cyclic features (often based on time, season, etc.) benefit from coding that takes account of their periodic nature. Otherwise adding an hour to an hour feature can have unexpected effects if the feature was assumed to be a simple number.

Categorical features are widespread in data mining, and need special treatment to be included in models that accept only numerical data. There are many options and pitfalls....

This notebook has examples of the following encoders in action:

  • cyclic encoding ("homegrown")
  • dummy and indicator coding for nominal variables (pandas and scikit-learn versions)
  • ordinal encoding for ordinal features (have categories but there is a natural ordering, from scikit-learn)
  • label encoding for categorical targets (DO NOT USE on categorical features)

Some more advanced encoders are mentioned in passing. For example, a binary encoder might suit the case of a nominal feature with a large number of categories, which would otherwise add a large number of dummy features to the model.

The notebook also shows an example of encoders participating in a (scikit-learn) machine learning pipeline, which is another advanced topic that will receive attention in Data Mining 2.

The (generated) dataset supporting the notebook should be downloaded and placed in a data subdirectory of the folder where the notebook is placed.
